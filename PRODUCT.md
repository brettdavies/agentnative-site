# PRODUCT.md: agentnative-site channel

Channel-specific product + design context for the **site channel** of agentnative. Loaded by `/impeccable` (the
`load-context.mjs` script renames legacy `.impeccable.md` to this filename) and read by humans editing the site.

3. **Visual system — [`DESIGN.md`](DESIGN.md).** Deep design system: OKLCH palette, type scale, code-block treatment,

## Channel: site

The site is a static site generator + Cloudflare Worker shell. It ships HTML and markdown of the same content from the
same paths via `Accept:` content negotiation, plus `/llms.txt`, `/llms-full.txt`, and `.md` URL suffixes. Visual system
decisions (palette, typography stack, code-block treatment, OG image) are site-channel-only. Voice and the universal
anti-patterns come from `BRAND.md`.

The site's job-to-be-done: tell the reader, authoritatively and in one place, what it means for a CLI to be agent-native
and link to the linter that measures it.

## Audience (narrowed from BRAND.md)

- **Humans** evaluating or adopting the standard. Most arrive from a Show HN post or a social share. Many maintain CLIs
  themselves and want to know, in under 60 seconds, what the standard requires, whether their tool already conforms, and
  how to install the linter to check. They read on laptops and larger phones during work, in a context where they decide
  whether to take the standard seriously.
- **AI agents** consuming the spec programmatically via `/llms.txt`, `/llms-full.txt`, `.md` URL suffixes, and `Accept:
  text/markdown` content negotiation. Their UX is "does the markdown come back clean, are anchors stable, is the
  structure predictable across versions." Decisions that make the HTML prettier at the cost of the markdown channel are
  regressions.

Not primary audiences: managers deciding budget, procurement, the general public.

## Visual system

**Theme.** Both modes, user-toggleable. `prefers-color-scheme` as the default, explicit `[data-theme]` override,
`system` as a third state. Dark mode is deliberately designed, not inverted (see `DESIGN.md` §4.2).

**Palette posture.** Cool-neutral base (hue 250), one accent in the same hue family, three semantic warm accents for
MUST / SHOULD / MAY. Inline keyword color only; block-level variants deferred to live-site iteration. OKLCH throughout
with sRGB hex fallbacks. Full methodology and contrast verification in `docs/research/design/color-analysis.md`.

**Typography (settled).** Body and display: Pangram Pangram's Uncut Sans (OFL). Code: GitHub Next's Monaspace Xenon
(OFL). Loading strategy and full spec in `DESIGN.md` §4.3; preview at
`docs/research/design/must-should-may-preview.html`. Future typography changes re-run `/typeset` rather than propose
fonts from memory; the executable list of fonts the channel rejects lives in
[`styles/site/README.md`](styles/site/README.md).

**Banned aesthetic patterns.** Dashboard / admin-panel patterns, product-landing patterns, AI-slop palette tropes, and
"serious documentation" templating shapes are out. The literal enforced bans (and the rationale per category) live in
[`styles/site/README.md`](styles/site/README.md). The categories the site rejects:

- Dashboard / admin tropes (equal-tile card grids, sparkline decorations, side-stripe borders on callouts).
- Product-landing tropes (oversized hero sections, gradient backgrounds, big CTA buttons, glassmorphism, animated scroll
  reveals, hover-scale transforms).
- AI-slop palette (cyan/purple gradients on dark backgrounds, gradient text, neon accents, purple-to-blue hero washes).
- Documentation templating (icons above every section heading, identical cards for principles, monochrome vertical
  accents in a left sidebar).

**Reference points (what to borrow from).** `clig.dev` (single-column confidence, quiet accent, type rhythm),
`developers.cloudflare.com` (code-block treatment, copy UX, sidebar TOC that tracks scroll, dark-mode intentionality),
`rust-lang.org/book` (inline `code` and block code share a visual family; generous code padding), `simonwillison.net`
(plain-HTML-feel can carry real authority; visible author-hand without marketing chrome), `anthropic.com/claude/docs`
(dense type, code as first-class).

## Channel-specific principles

The universal voice anchors (concrete-before-abstract, show-then-tell, no filler adjectives) come from `BRAND.md`.
Site-specific surface registers (1a homepage lede vs 1b principle page, 2 task pages, 3 markdown twins) live in
[`docs/research/VOICE.md`](docs/research/VOICE.md). The site channel adds:

1. **The markdown source is the source of truth.** Any design decision that would force the `text/markdown` channel to
   carry presentational markup loses to the channel that keeps it clean. The HTML render is derivative.
2. **Authority through precision, engagement through detail.** The site's credibility comes from measured contrast,
   stable anchors, and RFC-grade language. The site's stickiness comes from details a reader only notices because
   they're reading slowly: the tabular-numeral version-and-date in the footer, a small-caps RFC 2119 keyword, a
   code-block corner-label that does not shout.
3. **Restraint is not austerity.** One distinctive, considered choice per dimension (one typeface family with character,
   one accent, one motion idiom) — then stop.
4. **Earn every element.** Each typeface, each animation, each interactive widget MUST pass "what would the site lose
   without this?" If the answer is "nothing," cut it. The total homepage budget is 2 MB; JS is fine. The taste budget is
   tight.

## Tech stack and scope

- **Stack (decided).** Static site generation; plain HTML from markdown via a 200-line Node build; Cloudflare Worker (80
  lines) for routing, `Accept: text/markdown`, `.md` URL suffix, `Link` and `X-Llms-Txt` response headers, and
  `X-Robots-Tag: noindex` on the markdown variant. See `DESIGN.md` §3.
- **Scope (decided).** Index plus 7 principle anchors on one page, plus `/audit` and `/about`. Nine surfaces total.
  `DESIGN.md` §3.1 locks SSG as non-negotiable.
- **Accessibility.** WCAG 2.1 AA and APCA Lc ≥ 60 body minimum both pass in both modes, verified in
  `docs/research/design/color-analysis.md`. Smaller sizes or lower-contrast weights MUST be re-checked on any typography
  change.
- **JS posture.** JS is fine; total homepage budget is 2 MB. Use a library if it earns its place. Native Clipboard API
  is fine; framework runtimes still need justification.
