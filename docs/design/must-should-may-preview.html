v0.1 · 2026-04-14

P1 · Non-Interactive by Default

CLI tools that block on interactive prompts are invisible to agents. The agent hangs, the user sees nothing, and the operation times out silently. A tool MUST support a non-interactive path gated by --no-interactive (or the equivalent environment variable) and SHOULD auto-detect TTY presence via std::io::IsTerminal. Rich interactive affordances MAY appear when a TTY is present and --no-interactive is not set.

Evidence
#[derive(Parser)]
struct Cli {
    /// Suppress all interactive prompts.
    #[arg(long, env = "TOOL_NO_INTERACTIVE", global = true,
          value_parser = FalseyValueParser::new())]
    no_interactive: bool,
}

§4.7 · 7a — plain bold (baseline)

No keyword color

Tools MUST run non-interactively when stdin is not a TTY. The --no-interactive flag MUST gate every prompt at the entry point.

Tools SHOULD auto-detect non-interactive contexts via TTY detection (IsTerminal) and suppress prompts when stderr is not a terminal.

Tools MAY offer rich interactive experiences (spinners, progress bars) when a TTY is detected.
